# aus-temp-climatology

Interactive temperature climatology comparison tool for Australian NEM sites.

Compares current seasonal temperature forecasts against a user-defined historical window (1996‚Äì2025) for four Bureau of Meteorology observation sites: **Adelaide West Terrace, Melbourne Olympic Park, Bankstown, and Archerfield.**

Built by the MetraWeather Australian Consult Team.

---

## What it does

- Loads six months of seasonal Tmax / Tmean / Tmin forecasts
- Plots each forecast month against the distribution of historical observations for the same calendar month
- A dual-handle slider lets you set any start and end year within 1996‚Äì2025, recomputing the climatology instantly
- Box-and-whisker chart per city shows IQR, min‚Äìmax range, mean, and where the forecast lands
- Optional overlay of the previous month's forecast to visualise forecast drift
- Flags station changeover years (Adelaide 2017, Melbourne 2013) when the selected window crosses them

## Setup ‚Äî first time

Follow these steps in order.

---

### Step 1 ‚Äî Create the GitHub repo

1. Go to [github.com](https://github.com) and log in
2. Click **+** (top right) ‚Üí **New repository**
3. Name it: `aus-temp-climatology`
4. Set visibility to **Public**
5. Tick **Add a README file**
6. Click **Create repository**

---

### Step 2 ‚Äî Generate obs_data.csv

1. Copy `prepare_obs_data.py` into your Climate_Slider folder:
   ```
   C:\Users\tconstable\OneDrive - MetService\Desktop\Projects\Climate_Slider\
   ```

2. Open a terminal (`Win + R` ‚Üí type `cmd` ‚Üí Enter)

3. Navigate to Climate_Slider:
   ```
   cd "C:\Users\tconstable\OneDrive - MetService\Desktop\Projects\Climate_Slider"
   ```

4. Run the script:
   ```
   python prepare_obs_data.py
   ```

5. You should see output like:
   ```
   Processing Adelaide...
     Loaded 1996-2016: 252 month-rows
     Loaded 2017-2025: 108 month-rows
   ...
   ‚úì Done. 1440 rows written to: ...\Climate_Slider\obs_data.csv
   ```

6. Check that `obs_data.csv` now exists in the Climate_Slider folder

---

### Step 3 ‚Äî Upload files to GitHub

1. Go to your repo: `github.com/2manytims/aus-temp-climatology`
2. Click **Add file** ‚Üí **Upload files**
3. Drag and drop all four files:
   - `index.html`
   - `obs_data.csv`
   - `AusSeasonalTempPlusPrev.csv`
   - `prepare_obs_data.py`
4. Add a commit message: `Add initial data and tool`
5. Click **Commit changes**

---

### Step 4 ‚Äî Enable GitHub Pages

1. In your repo, click **Settings** (top menu)
2. Click **Pages** (left sidebar)
3. Under **Source**, select **Deploy from a branch**
4. Set branch to `main`, folder to `/ (root)`
5. Click **Save**
6. After a minute or two, your live URL will appear:
   ```
   https://2manytims.github.io/aus-temp-climatology/
   ```

---

### Step 5 ‚Äî Test it

- Drag the sliders to different year ranges ‚Äî charts should update instantly
- Toggle between Tmax, Tmean, Tmin
- Tick "Show previous month's forecast" ‚Äî orange dots should appear

## Updating the forecast each month

1. Go to your repo on GitHub
2. Click on `AusSeasonalTempPlusPrev.csv`
3. Click the **...** menu ‚Üí **Delete**, commit the deletion
4. Then **Add file** ‚Üí **Upload files**, drag in the new file, commit

## Data sources

- **Temperature observations:** Bureau of Meteorology daily climate data
  - Adelaide: Kent Town (023090) 1996‚Äì2016, West Terrace (023000) 2017‚Äì2025
  - Melbourne: Regional Office (086071) 1996‚Äì2012, Olympic Park (086338) 2013‚Äì2025
  - Bankstown: single station, full record
  - Archerfield: single station, full record
- **Seasonal forecasts:** MetraWeather internal NEM temperature forecast product
